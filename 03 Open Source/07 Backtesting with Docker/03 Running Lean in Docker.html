Before we can go any further, we need to download Lean's source from its GitHub repository located here. To do this, you'll need to have git installed. If you don't already have it, you can follow the installations at git's website found here.

To clone Lean's repository, open a terminal and run: 

    git clone https://github.com/QuantConnect/Lean.git
This downloads the repository to your current directory, under a new directory named /Lean by default. If you wish to download Lean into a different folder, you can append a directory name onto the back half of the command like so: 
    git clone https://github.com/QuantConnect/Lean.git /path/of/your/desired/directory/
where /path/... is the location of the directory you want Lean to be downloaded into.

Next, to enter that new directory for the next steps: 

    cd /path/of/directory/from/the/previous/steps

    # or, if you're using the default location:

    cd Lean

Now, Lean ships with a script named run_docker which does exactly as the name suggests: it creates a docker container, builds Lean, runs it, and outputs the restults in the results folder, which is, by default, ~/Lean/Results (or in the Results folder of whatever directory you downloaded Lean's source to).

To run the aforementioned script: 

    # on linux/mac:
    sudo ./run_docker.sh

    # on windows:
    C:\path\to\the\lean\directory\run_docker.bat
(sudo is required if your user doesn't have superpuser permissions, because docker needs these to run)
